# 前端开发面试标准答案（真题实战版）

## 三、JavaScript 核心 (初级/资深)

### Q12: [资深] 如何保证前端与 LLM 交互时的隐私与安全？
**A:** 
1. **脱敏处理**：在发送到 LLM 之前，前端或中间层应使用正则/NLP 识别并遮蔽身份信息 (PII)。
2. **鉴权透传**：通过 JWT/Session 将用户身份标识传递给后端。
3. **基于 RAG 的权限控制 (ACL)**：在检索阶段，VectorDB 应根据用户权限过滤文档块（Metadata Filter），确保 LLM 仅基于该用户有权访问的上下文回答。
4. **CORS/CSP**：防止模型生成的内容中包含恶意脚本进行 XSS 攻击。

## 四、前端框架与工程化 (Vue/React)

### Q15: [资深] 什么是低代码（Low-Code）表单引擎？如何设计？
**A:** 
- **核心组件**：1. **配置面板** (Canvas/Editor)。2. **渲染器** (Renderer)。3. **属性面板** (Properties Panel)。
- **实现方案**：1. 定义标准的 JSON Schema 协议描述表单项。2. 使用递归组件结构渲染嵌套表单。3. 通过 `Computed` 或 `Watch` 处理表单项间的联动。4. 性能优化：使用 `v-memo` 或 `memo` 仅对变动的表单项重绘。

## 五、项目经历与场景深挖 (STAR 法则)

### Q17: 如何杜绝/识别面试 AI 辅助作弊？
**A:** 
1. **Visibility API**：监测 `visibilitychange` 事件，记录候选人切屏次数。
2. **复制/粘贴锁定**：在编辑器中拦截 `paste` 事件。
3. **输入流特征分析**：通过 JS 监听 `keydown/keyup` 间隔。AI 辅助通常是“瞬间输入大量字符”，而人类输入具有自然的随机波动节律。
4. **面试官 Callback (纠错)**：像真题里说的，面试官通过追问或回调之前的回答来验证逻辑的一致性。

### Q18: 提升 AI 生成结果的准确度感知？
**A:** 
1. **引用标注 (Citations)**：在 AI 回复中对每个事实点打上引用标记，鼠标悬停显示原始文档出处。
2. **置信度评分 (Confidence Score)**：根据检索结果的相关性分值，向用户展示本次回答的“准确度预估”。
3. **多模型对比**：提供“AI 纠错”按钮，允许用户调用更强的模型（如 GPT-4）对当前答案进行核对。
